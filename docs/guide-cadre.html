Piloter les stages étudiants : planning, heures, validations

Sommaire 1. Principe général 2. Où se passe quoi 3. Établir le planning 4. Comprendre les heures 5. Jours fériés 6. Valider les déclarations 7. Vos coordonnées 8. Planning imprimable & évaluations

1. Principe général

L'étudiant dispose d'un espace web où il consulte son planning et déclare ses écarts d'heures. Toutes les données vivent dans ce document Grist : le site web n'est qu'une fenêtre par-dessus.

Étudiant → Site web (code anonymat) → Proxy sécurisé → Document Grist (vous)

L'étudiant n'a pas de compte Grist. Il s'identifie par son code anonymat (1re lettre du prénom + date de naissance JJMMAA + 1re lettre du nom).

2. Où se passe quoi (tables Grist)

TableRôle
LISTE_DES_ETUDIANTSFiche étudiant. Créée automatiquement à l'entrée en stage.
PERIODES_DE_STAGEUne ligne par stage (service, dates, niveau, tuteur, heures à faire/faites).
PLANNING_HEBDOUne ligne par semaine. C'est ici que vous saisissez le planning (codes du lundi au dimanche).
Sortie_de_stageLes déclarations de l'étudiant (heures sup, retards, sorties). C'est ici que vous validez.
CODES_HORAIRESRéférentiel des codes (M, S, N, JF…) et de leurs horaires.
SERVICES / UTILISATEURSServices et cadres responsables.

3. Établir le planning d'un étudiant

  1. Ouvrez la page de gestion et sélectionnez l'étudiant / sa période.
  2. Les semaines sont générées automatiquement à l'inscription (une ligne par semaine de stage). Sinon, utilisez le bouton « Générer les semaines ».
  3. Dans PLANNING_HEBDO, renseignez pour chaque jour un code horaire (M, S, N, R, JF…).

Le total d'heures de la semaine et de la période se calcule tout seul. L'étudiant voit le planning en lecture seule.

4. Comprendre le décompte des heures

NotionComment c'est calculé
À réaliser (A_FAIRE)35 h × nombre de semaines de stage. Renseigné automatiquement à l'inscription ; vous pouvez le modifier dans la période.
Effectuées (FAIT)Somme des heures du planning + des déclarations validées.
SoldeEffectuées − À réaliser. Négatif = il reste des heures à faire.
Les heures d'un code = sa durée + son éventuel ajustement, uniquement si le code « compte pour le stage » (colonne Compte_stage dans CODES_HORAIRES).

5. Jours fériés

Les jours fériés sont listés dans la table JOURS_FERIES. Règle appliquée (conforme à l'arrêté du 31 juillet 2009) :

  • Le férié est accordé : il est déduit du volume à réaliser (A_FAIRE = 35 h/semaine − 7 h par férié). Un férié non travaillé ne crée donc pas de dette.
  • Un férié travaillé compte ses heures normales (plus de ×2) et ouvre un jour de récupération, affiché sur la fiche de l'étudiant.
A_FAIRE est recalculé automatiquement à l'inscription. Pour un cas particulier, ajustez la valeur A_FAIRE directement dans la période. Le décompte compte tous les fériés tombant dans la période de stage.

6. Valider les déclarations d'heures

Quand un étudiant déclare des heures, la ligne apparaît dans Sortie_de_stage avec la case Validé décochée.

  • Tant qu'elle n'est pas validée, elle s'affiche « En attente » côté étudiant et ne compte pas dans ses heures.
  • Cochez Validé pour la prendre en compte : elle devient Validé chez l'étudiant et alimente le total.
  • Une déclaration validée ne peut plus être supprimée par l'étudiant.

Les trois types de déclaration

Type (colonne Motif)Effet
Rattrapage / Heures supplémentairesAjoute les heures.
RetardDéduit les heures (la formule reconnaît « Retard »).
Sortie de stageAjoute les heures si « Compte en stage » est coché, sinon 0.

La précision libre saisie par l'étudiant se trouve dans la colonne Motif ou Commentaire.

Ne renommez pas le libellé d'une colonne en pensant que c'est cosmétique : Grist régénère son identifiant technique et peut casser les formules. En cas de doute, demandez.

7. Vos coordonnées de cadre

Le nom, le mail et le téléphone du cadre s'affichent à l'étudiant (sur sa fiche de stage et dans le message d'inscription). Ils viennent de la table UTILISATEURS (via SERVICES.Cadre_ref).

Pensez à renseigner la colonne Téléphone de la table UTILISATEURS : tant qu'elle est vide, seul le mail s'affiche.

8. Planning imprimable & évaluations

  • Planning imprimable : une mise en page prête à imprimer/signer (récapitulatif des semaines, heures, sorties, rendez-vous formateur, signatures).
  • Évaluations de stage : les questionnaires de satisfaction sont collectés dans EVALUATION_STAGE_ETUDIANT (scores par thème, note globale, recommandation, commentaires). Un tableau de bord d'analyse est en préparation.
